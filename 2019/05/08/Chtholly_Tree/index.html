珂朵莉树节点的定义

珂朵莉树是基于$std ::set$的数据结构，其声明如下

可以发现，每一个节点存储的，是一个连续的等值区间。（注意：点值的存储必须添加$mutable$关键字， 否则无法进行区间修改操作）。

由于珂朵莉树是构造在$std :: set$上的，自然不能用下标访问。所以我们先宏定义一个迭代器，以免把时间都花费在敲迭代器上。

对于区间操作，珂朵莉树需要将一段区间分裂出来，在这里我们引入第一个函数

端点分裂

这个操作能将指定点所在位置的区间分裂成两段，以便进行接下来的其他操作。
这里有一个小细节：最终返回的迭代器是由$set$的$insert$函数返回的。因为该函数的返回值的类型为$pair<iterator,\ bool>$,其中第一个是节点插入位置的迭代器，第二个标记是否插入成功（因为$set$对于相等的节点并不会重复插入）。

区间操作

注意！取出区间时应先split右端点，否则可能会RE

因为在$split$操作中，若左右端点在珂朵莉树的同一节点内，则先split左端点可能会导致右端点split时，左端点所在的区间被$erase$，则在遍历时会因访问无效内存出错。（这也是在其他时候使用指针和迭代器是应注意的，切忌使用过期的地址）

显然珂朵莉树是通过暴力枚举进行的区间操作：将每段连续等值区间分别取出，然后依次操作。

更显然这样操作时间复杂度并不优秀。

然而很多人可能是听说珂朵莉树能吊打标程才打算学习珂朵莉树的（本人是因为珂朵莉树的名字才学的珂朵莉树）。

所以接下来请出珂朵莉树的核心操作：

区间赋值

正是这个操作拯救了珂朵莉树的时间复杂度。

对于区间赋值操作，珂朵莉树只需要将该区间清空，再插入一个新节点就可以了。

所以，在看到数据结构题有区间赋值操作，又想不到满分算法时，珂朵莉树往往可以骗到很多分

前提是出题人没有可以卡珂朵莉树（只要在数据里不加入区间赋值就好了）
